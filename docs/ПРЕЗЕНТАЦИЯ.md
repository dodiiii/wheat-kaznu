# 🌾 IoT-мониторинг пшеничных полей — сценарий презентации

Документ-шпаргалка для показа дашбордов **хозяину предприятия**. По каждой панели:
**Что** видно · **Зачем** это агроному/владельцу · **Как** считается · **Какое решение** помогает принять.

> 🎞️ **Красивая веб-версия для показа заказчику** (с разбором каждой панели и IoT-роадмапом):

>

**Ключевой тезис для заказчика:** сегодня система работает на **реальных исторических данных Шелекской долины (Алматинская обл.)**
(Open-Meteo / ERA5-Land по координатам полей) — это демонстрация полного сезона «как если бы система
стояла на полях с весны». В production источник меняется на **ваши IoT-датчики в реальном времени**,
при этом **ни один дашборд не переделывается**: датчики пишут в те же ряды InfluxDB, где сейчас лежит история.

## 0. Как устроена система (вступление на 1 минуту)

- **Регион:** Алматинская область, Казахстан — Шелекская долина, Енбекшиказахский район (предгорье Заилийского Алатау, каштановые почвы). 5 полей вдоль долины — одна климатическая зона.
- **Поля:** Шелек (120 га), Байсеит (95 га), Малыбай (150 га), Ават (110 га), Каратурык (130 га). Координаты ~43.5° с.ш., 78.2° в.д.
- **Сезон:** 2025-05-01 → 2025-09-30 (153 дня), шаг — 1 сутки (для техники — 30 минут).
- **Источники данных:**
  - Погода и почвенная динамика — **реальные** из Open-Meteo по координатам полей (модель ERA5-Land, лицензия CC BY 4.0).
  - Статика почвы (pH, углерод, азот, гранулометрия) — по типичным значениям **SoilGrids** для каштановых почв Шелекской долины.
  - GDD, фаза BBCH, NDVI, прогноз урожая, вредители, техника — **рассчитаны/синтезированы** и привязаны к тем же координатам и датам.
- **Стек:** InfluxDB (база временных рядов) + Grafana (визуализация), всё поднимается одной командой `docker compose up`.
- **Принцип «один вопрос — один дашборд»:** каждый экран отвечает на свой управленческий вопрос и построен по схеме **① ИТОГ → ② ПРИЧИНЫ → ③ ДЕТАЛИ** (от общего к частному).

**Ключевая фишка для демонстрации:** вверху каждого дашборда — фильтр **«Поле»**. При его переключении **все панели и карта мгновенно перестраиваются** на выбранное поле (карта сама зумится на его контур).

> ⚠️ Сегодня — реальные **исторические** данные Шелекской долины (демонстрация полного сезона).
> В production источник меняется на **ваши IoT-датчики в реальном времени** — дашборды не переделываются
> (датчики пишут в те же ряды InfluxDB). См. раздел 6 «Дорожная карта».

### Легенда цветов (единая по всем панелям)
| Цвет | Значение |
|------|----------|
| 🔵 Синий | норма/избыток влаги, прохладно |
| 🟢 Зелёный | оптимум — всё хорошо |
| 🟠 Оранжевый | внимание, приближение к порогу |
| 🔴 Красный | критично — нужно действовать |

> Цвет — это не украшение: пороги заданы по агрономическим ЭПВ (экономическим порогам вредоносности) и водно-температурным нормам яровой пшеницы.

## 1. 🌾 Обзор хозяйства (Farm Overview)
**Вопрос дашборда:** «Что происходит на поле и где нужно вмешаться?»
**Структура (сверху вниз):** ① ИТОГ (что делать) → ② ПРИЧИНЫ (показатели + карта) → ③ ДЕТАЛИ (динамика за сезон).
Это «титульный экран» для владельца: заходит утром — за 10 секунд видит статус.

### 📍 Карта полей (Geomap)
- **Что:** поле на реальной карте OpenStreetMap с маркером и последней средней температурой.
- **Зачем:** пространственная привязка — владелец видит, о каком именно участке идёт речь.
- **Как:** координаты (lat/lon) берутся из данных о погоде, последнее значение температуры. Режим карты — `fit`: при смене поля она **автоматически центрируется и зумится** на его контур.
- **Решение:** быстрая навигация «где это поле», основа для логистики и объезда.

### 🌡️ Средняя температура (Stat)
- **Что:** средняя температура воздуха за сезон, °C.
- **Зачем:** базовый драйвер роста и испарения влаги.
- **Как:** среднее по всем суточным значениям `temperature_mean` (ERA5).
- **Пороги:** <10 синий (холодно) · 10–25 зелёный (оптимум) · 25–35 оранжевый · >35 красный (тепловой стресс).
- **Решение:** оценка теплообеспеченности сезона, риск жаровки в налив.

### 🌧️ Всего осадков (Stat)
- **Что:** сумма осадков за сезон, мм.
- **Зачем:** вода — ключевой фактор урожая на богаре в предгорьях Алматинской области.
- **Как:** сумма суточных `precipitation`. ~250 мм за сезон считается достаточным.
- **Пороги:** <100 красный (засуха) · 100–200 оранжевый · 200–400 зелёный · >400 синий (избыток).
- **Решение:** нужен ли полив/влагосбережение, реалистичность плана по урожаю.

### 🌱 Средний NDVI (Stat)
- **Что:** текущий индекс вегетации (0–1) — «зелёность»/биомасса посева.
- **Зачем:** интегральный показатель здоровья растений; коррелирует с урожаем.
- **Как:** последнее значение `ndvi` (модельная кривая от накопленного тепла GDD).
- **Пороги:** <0.3 красный (слабый посев) · 0.3–0.5 оранжевый · 0.5–0.7 зелёный · >0.7 отличный.
- **Решение:** ранний сигнал угнетения посева до того, как это видно глазом.

### 💧 Влажность почвы 0–7 см (Stat)
- **Что:** влага в верхнем (посевном) слое, % объёма.
- **Зачем:** от неё зависят всходы, работа удобрений, стартовый рост.
- **Как:** последнее значение `moisture` на глубине 0–7 см (ERA5-Land).
- **Пороги:** <15% красный (сухо) · 15–25% оранжевый · 25–40% зелёный · >40% синий (переувлажнение).
- **Решение:** можно ли вносить сухие удобрения, стоит ли отложить обработку почвы.

### 📈 Прогноз урожайности (Stat)
- **Что:** ожидаемая урожайность, т/га.
- **Зачем:** это деньги — прямой ориентир для владельца.
- **Как:** эмпирическая модель `урожай = база × min(1, GDD/1800) × min(1, осадки/250)`.
- **Пороги:** <2 красный · 2–3 оранжевый · 3–4 зелёный · >4 отличный.
- **Решение:** планирование выручки, логистики уборки, продаж зерна.

### ⚠️ Очаги вредителей (Stat)
- **Что:** число вспышек с тяжестью ≥6 за сезон.
- **Зачем:** индикатор фитосанитарного давления и риска потерь.
- **Как:** счётчик событий `outbreak == 1` (тяжесть ≥6 по шкале 1–10).
- **Пороги:** <10 зелёный · 10–30 оранжевый · >30 красный.
- **Решение:** нужен ли усиленный мониторинг и бюджет на СЗР.

### 🚦 Уровень угрозы · 📋 Рекомендуемое действие · 💡 Агрономическое обоснование (Stat)
- **Что:** «умный светофор» — итоговый вердикт по полю на сегодня + конкретное действие + пояснение почему.
  - 🟢 НОРМА / 🟡 ВНИМАНИЕ / 🔴 ОПАСНОСТЬ.
  - Пример: «Внесение фунгицидов 🛑» + «Обнаружена бурая ржавчина (тяжесть 7/10), риск перезаражения стебля».
- **Зачем:** превращает данные в **готовую рекомендацию** — это главная «продающая» ценность системы для владельца.
- **Как:** движок правил связывает фазу роста, влажность почвы, погоду и вредителей в одно решение (`agronomy_insights`).
- **Решение:** что конкретно делать на поле сегодня — без агронома-аналитика.

### 🌡️ Температура / 🌧️ Осадки / 🌱 NDVI по полям (Time series)
- **Что:** динамика за сезон линиями/столбцами.
- **Зачем:** тренд важнее одной цифры — видно волны тепла, дожди, набор и спад «зелёности».
- **Как:** суточная агрегация (`aggregateWindow 1d`); в легенде — среднее/сумма/макс.
- **Решение:** сопоставление «погода → реакция посева», объяснение отклонений.

### 📊 Сводка по полям (Table)
- **Что:** таблица с последними GDD, NDVI и прогнозом урожая; урожай — цветной мини-гейдж.
- **Зачем:** компактное сравнение «одним взглядом».
- **Решение:** приоритизация — куда направить ресурсы в первую очередь.

### 📈 GDD накопление (Time series)
- **Что:** накопленная сумма активных температур за сезон.
- **Зачем:** GDD — «биологические часы» пшеницы: по нему предсказываются фазы и сроки уборки.
- **Как:** нарастающий итог `gdd_cumulative`.
- **Решение:** прогноз наступления фаз и даты уборки.

## 2. 🌤️ Метео и почва (Meteo & Soil)
**Вопрос дашборда:** «Какие условия сложились — в воздухе и в почве?»
**Структура:** ① ВОЗДУХ (погодный фон) → ② ПОЧВА: ДИНАМИКА (корневая зона) → ③ ПОЧВА: ПАСПОРТ (свойства).
Это доказательная база: показываем, что рекомендации опираются на реальные ERA5-данные.

### 🌡️ Температура воздуха + Влажность (двойная ось Y)
- **Что:** температура (min/mean/max, левая ось °C) и относительная влажность (правая ось %).
- **Зачем:** связка «жара + сухость» = стресс; «тепло + высокая влажность» = риск грибных болезней.
- **Как:** суточные значения ERA5; две оси, потому что единицы разные.
- **Решение:** прогноз стресса и вспышек болезней.

### 🌧️ Осадки + 💨 Скорость ветра (двойная ось Y)
- **Что:** осадки столбцами (мм) и максимальный ветер (м/с).
- **Зачем:** дожди пополняют влагу; сильный ветер — нельзя опрыскивать (снос) и растёт испарение.
- **Решение:** выбор «окон» для опрыскивания и полевых работ.

### ☀️ Солнечная радиация + 🌿 Эвапотранспирация ET₀ (двойная ось Y)
- **Что:** приход солнечной энергии и эталонное испарение ET₀ (мм).
- **Зачем:** ET₀ = сколько влаги поле теряет; вместе с осадками даёт водный баланс.
- **Решение:** расчёт дефицита влаги и потребности в поливе.

### 💧 Влажность почвы по глубинам (Time series)
- **Что:** влага на трёх горизонтах — 0–7, 7–28, 28–100 см.
- **Зачем:** верх сохнет первым, но у пшеницы глубокие корни — важно видеть весь профиль.
- **Пороги:** линия <15% (красная зона) = дефицит влаги.
- **Решение:** оценка запаса влаги для растения на разной глубине, сроки полива.

### 🌡️ Температура почвы по глубинам (Time series)
- **Что:** прогрев почвы на 3 глубинах.
- **Зачем:** от температуры посевного слоя зависят всходы и активность корней; глубокие слои — тепловой «буфер».
- **Решение:** определение сроков сева, прогноз старта вегетации.

### 🧪 Статические свойства почвы (Bar gauge)
- **Что:** pH (у полей 7.6–8.1 — слабощелочные каштановые почвы), органический углерод (11–16 г/кг), азот (1.1–1.6 г/кг).
- **Зачем:** «паспорт плодородия» поля — потенциал и потребность в удобрениях. Для каштановых почв предгорий характерны умеренный гумус и щелочная реакция.
- **Как:** фиксированные значения по SoilGrids для конкретного поля Шелекской долины.
- **Решение:** расчёт норм удобрений и мелиорации (при высоком pH — подкисляющие формы азота).

### 🏔️ Гранулометрический состав (Pie chart)
- **Что:** доли песка / пыли / глины.
- **Зачем:** определяет влагоёмкость и как быстро почва отдаёт/держит воду.
- **Решение:** стратегия орошения и обработки под тип почвы.

## 3. 🌱 Факторы роста (Growth Factors)
**Вопрос дашборда:** «На какой стадии посев и куда движется урожай?»
**Структура:** ① ИТОГ (фаза + прогноз) → ② ГЛАВНЫЙ ФАКТОР (тепло/GDD) → ③ ДЕТАЛИ (связи факторов).
Связываем тепло → фазу → «зелёность» → прогноз урожая.

### 📈 Накопление GDD + Фаза роста (двойная ось Y)
- **Что:** накопленный GDD (линия) и суточный прирост GDD (столбцы).
- **Зачем:** показывает «биологический возраст» пшеницы и темп развития.
- **Решение:** прогноз наступления следующих фаз.

### 🌾 Текущий GDD (Stat)
- **Что:** сумма активных температур на сегодня.
- **Пороги:** <500 · 500–1200 · 1200–1700 · >1700 (созревание).
- **Решение:** быстрый ответ «сколько тепла уже набрано».

### 📊 Фаза BBCH (Stat)
- **Что:** текущая фаза по международной шкале BBCH (всходы → кущение → трубка → колошение → налив → спелость).
- **Зачем:** каждая фаза = свой набор работ и рисков; язык, понятный любому агроному.
- **Как:** маппинг накопленного GDD на порог фазы.
- **Решение:** привязка агроопераций к правильной фазе.

### 🌾 Прогноз урожайности (Gauge)
- **Что:** ожидаемая урожайность т/га в виде стрелочного индикатора.
- **Решение:** ключевой бизнес-ориентир, обновляется по ходу сезона.

### 🌿 Текущий NDVI (Stat)
- **Что:** индекс вегетации сейчас (0–1).
- **Решение:** мгновенная оценка состояния посева.

### 🌿 NDVI + 💧 Влажность почвы 0–7 см (двойная ось Y)
- **Что:** «зелёность» и влага верхнего слоя вместе.
- **Зачем:** проверка гипотезы «падает влага → падает NDVI» — виден водный стресс.
- **Решение:** ранняя диагностика засушливого угнетения.

### 🌡️ GDD vs Температура (двойная ось Y)
- **Что:** суточный GDD и средняя температура.
- **Зачем:** наглядно показывает, что прирост GDD напрямую следует за теплом (валидация модели).
- **Решение:** доверие к прогнозу фаз.

### 📈 Прогноз урожайности + GDD + NDVI (overlay)
- **Что:** три кривые на одном графике — как формируется урожай.
- **Зачем:** «история сезона» одним взглядом: тепло растёт → биомасса растёт → прогноз выходит на плато.
- **Решение:** объяснение владельцу, из чего складывается итоговая цифра.

### 🌡️ NDVI vs Температура + Осадки (overlay)
- **Что:** «зелёность» на фоне погоды.
- **Зачем:** видно, как жара и дожди отражаются на состоянии посева.
- **Решение:** связывание провалов NDVI с конкретными погодными событиями.

## 4. 🐛 Вредители и болезни (Pests & Diseases)
**Вопрос дашборда:** «Где и когда угроза — и что её разгоняет?»
**Структура:** ① ИТОГ (уровень давления) → ② КОГДА И КТО (картина сезона) → ③ ПОЧЕМУ (драйверы + журнал).
5 объектов: бурая ржавчина, септориоз, пшеничный трипс, зерновая совка, хлебная блошка.

### KPI-строка: ⚠️ Очаги (sev≥6) · 📊 Средняя тяжесть · 🌿 Сорняки % · 🔢 Всего событий
- **Что:** сводка фитосанитарного давления на поле.
- **Зачем:** быстрая оценка масштаба проблемы и потребности в СЗР.
- **Пороги (тяжесть 1–10):** ЭПВ ≈ 6; сорняки >20% — красная зона.
- **Решение:** нужна ли обработка и насколько срочно.

### 🔥 Тепловая карта тяжести (Heatmap)
- **Что:** матрица «день × тяжесть» — где темнее/ярче, там сильнее давление.
- **Зачем:** сразу видно периоды вспышек по сезону.
- **Решение:** планирование защитных обработок на пиковые окна.

### 📊 Тяжесть по типам (Time series)
- **Что:** динамика тяжести по каждому вредителю отдельно.
- **Зачем:** у каждого объекта своё «окно» (блошка — на всходах, ржавчина — позже); нужны разные препараты.
- **Решение:** выбор конкретного препарата под конкретный объект.

### 🔗 Тяжесть vs Влажность + GDD (двойная ось Y)
- **Что:** тяжесть поражения на фоне влажности воздуха и фазы (GDD).
- **Зачем:** доказывает причинно-следственную связь — «высокая влажность в колошение → ржавчина/септориоз».
- **Решение:** **прогноз** вспышек по погоде, а не постфактум.

### 📋 Журнал вспышек sev≥6 (Table)
- **Что:** список серьёзных событий: дата, тип, тяжесть.
- **Зачем:** документальная база для истории поля и разбора.
- **Решение:** отчётность и планирование севооборота/защиты на след. сезон.

## 5. 🚜 Техника (Equipment)
**Вопрос дашборда:** «Как отработала техника — покрытие, скорость, простои?»
**Структура:** ① ИТОГ (выработка) → ② ГДЕ (треки на карте) → ③ ДЕТАЛИ (режим + журнал).
Операции: сев, удобрение, опрыскивание, уборка. Машины: 2 трактора, опрыскиватель, комбайн.

### 📍 Треки техники (Geomap)
- **Что:** GPS-треки проходов по полю на карте.
- **Зачем:** контроль полноты обработки — видно пропуски и перекрытия.
- **Как:** точки lat/lon с шагом 30 мин; карта в режиме `fit` зумится на выбранное поле.
- **Решение:** контроль качества работ и расхода топлива/СЗР.

### KPI-строка: 📏 Покрытие га · 🏎️ Средняя скорость · 📊 Кол-во точек · 🔧 Операций
- **Что:** сводка по объёму и интенсивности работ.
- **Зачем:** проверка, что поле обработано полностью и на правильной скорости (качество сева/опрыскивания зависит от скорости).
- **Пороги:** скорость <5 км/ч красный (слишком медленно/простой), покрытие <50 га оранжевый.
- **Решение:** оценка выработки, выявление недоработок.

### 🏎️ Скорость техники (Time series)
- **Что:** скорость по времени.
- **Зачем:** провалы = простои/поломки; превышения = риск брака работ.
- **Решение:** контроль дисциплины и техсостояния.

### 📋 Статусы операций (State timeline)
- **Что:** полоса состояний: работа / простой / старт / финиш.
- **Зачем:** наглядная доля продуктивного времени против простоев.
- **Решение:** оптимизация логистики и графика смен.

### 📋 Журнал операций (Table)
- **Что:** лог: время, машина, операция, статус, скорость, покрытие.
- **Зачем:** первичный документ для учёта и разбора.
- **Решение:** отчётность, расчёт зарплат/ГСМ, аудит.

## 6. Дорожная карта: от истории к real-time IoT

Каждому показателю системы уже соответствует конкретное устройство. Схема данных не меняется —
датчик просто начинает писать в те же ряды InfluxDB.

| Показатели | Устройство на поле | Канал | Частота |
|---|---|---|---|
| Температура, влажность, осадки, ветер, радиация | Полевая метеостанция (1 на 2–3 поля) | GSM / LoRaWAN | каждые 10 мин |
| Влажность и t° почвы на 3 глубинах | Профильный капацитивный зонд (1–2 на поле) | LoRaWAN (10–15 км в степи) | каждые 30 мин |
| NDVI, состояние посева | Sentinel-2 (бесплатно) + дрон с мультиспектральной камерой | API / облёт | раз в 3–5 дней |
| Вредители и болезни | Феромонные ловушки с камерой + мобильное приложение агронома | GSM | ежедневно |
| Позиция, скорость, статус техники | GPS/ГЛОНАСС-трекер + CAN-шина (топливо, обороты) | GSM | каждые 10–30 сек |
| pH, гумус, азот, гранулометрия | Агрохимический анализ в лаборатории | ручной ввод | раз в 2–3 года |

**Дополнительно в real-time режиме:** алерты в SMS/Telegram при срабатывании порогов
(заморозок, критическая засуха, вспышка вредителя) — 🔴-статус доставляется владельцу
в момент события, а не при следующем открытии дашборда.

## 7. Что подчеркнуть заказчику (итог за 30 секунд)

1. **Согласованность.** Все таблицы связаны едиными ключами `timestamp / field_id / lat / lon` — одно поле, один сезон, одна климатическая зона.
2. **Реальные данные Шелекской долины + честный синтез.** Погода и почва — реальный архив ERA5-Land по координатам полей; остальное рассчитано по прозрачным агрономическим формулам (описаны в README).
3. **От общего к частному.** 5 дашбордов — воронка: хозяйство → метео/почва → рост → защита → техника.
4. **Не сырые данные, а решения.** Блок «Уровень угрозы → Рекомендация → Обоснование» превращает мониторинг в готовый план действий.
5. **Готовность к real-time.** Архитектура спроектирована под IoT с первого дня: подключение датчиков не потребует переделывать ни один экран — меняется только источник данных.
6. **Открытый стек без абонентской ренты.** Grafana + InfluxDB, открытые лицензии, запуск одной командой `docker compose up`. Система принадлежит хозяйству, а не вендору.
