Демо-набор дашбордов для мониторинга пшеничного поля на основе открытых данных.
Весь стек поднимается одной командой `docker compose up`.

## 🚀 Быстрый старт

```bash
# 1. Поднять InfluxDB + Grafana
docker compose up -d

# 2. Дождаться инициализации InfluxDB (~10 сек)
sleep 10

# 3. Установить зависимости Python
pip install -r scripts/requirements.txt

# 4. Загрузить данные (Open-Meteo → CSV → InfluxDB)
python scripts/ingest.py

# 5. Открыть Grafana
#    http://localhost:3000  (admin / admin)
```

### Только CSV (без InfluxDB)

```bash
python scripts/ingest.py --csv-only
# CSV-файлы сохраняются в data/
```

## 📡 Источники данных

| Тип данных            | Источник                          | Реальные / синтетические |
|-----------------------|-----------------------------------|--------------------------|
| Метео (температура, осадки, ветер, радиация, ET₀) | [Open-Meteo Historical API](https://open-meteo.com/en/docs/historical-weather-api) (ERA5) | ✅ Реальные |
| Почва — влажность и температура (динамика) | Open-Meteo ERA5-Land (hourly soil) | ✅ Реальные |
| Почва — статика (pH, C, N, гранулометрия) | [SoilGrids](https://soilgrids.org) — типичные для региона | ⚡ На основе реальных диапазонов |
| GDD, фаза BBCH | Расчёт по формуле GDD = Σ max(0, T_mean − 0°C) | 🔧 Рассчитаны из метео |
| NDVI | Синтетическая кривая по GDD (логистический рост → спад) | 🔧 Модель |
| Прогноз урожайности | Эмпирическая модель (GDD × осадки) | 🔧 Модель |
| Вредители / болезни | Синтез: вероятность коррелирована с влажностью, температурой, GDD | 🔧 Синтетические |
| Техника (GPS-треки) | Синтетические треки по контуру полей | 🔧 Синтетические |

## 🔧 Правило синтеза недостающих данных

1. **Общие ключи:** все таблицы привязаны к осям `timestamp`, `field_id`, `lat`, `lon`.
2. **Метео и почвенная динамика:** берутся напрямую из Open-Meteo API (ERA5-Land), агрегируются из часовых в суточные.
3. **Почвенная статика:** фиксированные значения на основе типичных SoilGrids-данных для каштановых почв Шелекской долины (Алматинская область).
4. **GDD:** `GDD_daily = max(0, T_mean − T_base)` (`T_base = 0°C` для яровой пшеницы). Кумулятивное.
5. **Фазы роста BBCH:** маппинг по порогам GDD (0→ прорастание, 120→ листья, 350→ кущение, 600→ выход в трубку, 900→ бутонизация, 1050→ колошение, 1200→ цветение, 1450→ налив, 1700→ созревание).
6. **NDVI:** сигмоидный рост до GDD≈1100, затем линейный спад. Шум ±0.02.
7. **Урожайность:** `yield = base × min(1, GDD/1800) × min(1, precip_cum/250)`. `base` ∈ [2.8, 4.0] т/га по полям.
8. **Вредители:** для каждого дня и типа вредителя вероятность появления модулируется влажностью (>70% → +8%), температурой (15–28°С → +5%), осадками (>5 мм → +4%), фазой (400–1400 GDD → +6%). Дополнительные бонусы по типам (ржавчина ↔ RH>80 + T>18, фузариоз ↔ precip>8 + GDD>900, etc).
9. **Техника:** GPS-точки генерируются как линейный проход по прямоугольнику поля. Операции: сев (день 0–5), удобрение (20–23), опрыскивание (45–47, 75–77), уборка (135–142).

## 📊 Дашборды

Каждый дашборд отвечает на **один главный вопрос** и построен по принципу
сторителлинга **«от общего к частному»**: сверху — итог, ниже — причины, в самом низу — детали.
Секции разделены заголовками-строками (① / ② / ③). Подробный разбор каждой панели —
в [docs/ПРЕЗЕНТАЦИЯ.md](docs/ПРЕЗЕНТАЦИЯ.md).

### 1. 🌾 Обзор хозяйства (Farm Overview)
**❓ Вопрос:** в каком состоянии хозяйство сегодня и где нужно вмешаться?
- **① ИТОГ** — уровень угрозы, рекомендуемое действие, агрономическое обоснование
- **② ПРИЧИНЫ** — карта полей (Geomap) + KPI: температура, осадки, NDVI, влажность почвы, урожайность, очаги вредителей
- **③ ДЕТАЛИ** — таймсерии температуры / осадков / NDVI / GDD, сводная таблица по полям

### 2. 🌤️ Метео и почва (Meteo & Soil)
**❓ Вопрос:** какие условия сложились на поле — в воздухе и в почве?
- **① ВОЗДУХ** — температура + влажность (2 оси), осадки + ветер (2 оси), радиация + ET₀ (2 оси)
- **② ПОЧВА: ДИНАМИКА** — влажность и температура по 3 глубинам (0–7, 7–28, 28–100 см)
- **③ ПОЧВА: ПАСПОРТ** — pH / органический углерод / азот (bar gauge), гранулометрия (pie)

### 3. 🌱 Факторы роста (Growth Factors)
**❓ Вопрос:** на какой стадии посев и куда движется урожай?
- **① ИТОГ** — текущий GDD, фаза BBCH, прогноз урожайности (gauge), текущий NDVI
- **② ГЛАВНЫЙ ФАКТОР** — накопление GDD + суточный GDD (2 оси)
- **③ ДЕТАЛИ** — NDVI + влажность, GDD vs температура, урожай+GDD+NDVI, NDVI vs погода

### 4. 🐛 Вредители и болезни (Pests & Diseases)
**❓ Вопрос:** где и когда угроза посеву — и что её разгоняет?
- **① ИТОГ** — очаги (sev≥6), средняя тяжесть, покрытие сорняками, всего событий
- **② КОГДА И КТО** — heatmap тяжести по дням, тяжесть по типам вредителей
- **③ ПОЧЕМУ** — тяжесть vs влажность + GDD, журнал вспышек (severity ≥ 6)

### 5. 🚜 Техника (Equipment)
**❓ Вопрос:** как отработала техника — покрытие, скорость, простои?
- **① ИТОГ** — покрытая площадь, средняя скорость, точки, операции
- **② ГДЕ** — Geomap с GPS-треками техники
- **③ ДЕТАЛИ** — скорость (time series), статусы (state timeline), журнал операций

## 📝 Практики визуализации

- ✅ **Один вопрос на дашборд** — каждый дашборд отвечает на свой вопрос
- ✅ **Template variable `field_id`** — переключение между полями
- ✅ **Двойные оси Y** — для разных единиц измерения на одном графике
- ✅ **Пороги и алерты** — критически низкая влажность почвы, высокая тяжесть вредителей
- ✅ **Text-панель** — источник и период данных на каждом дашборде
